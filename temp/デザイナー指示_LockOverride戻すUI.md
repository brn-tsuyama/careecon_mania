# デザイナー指示書: Lock & Override + 「戻す」UI
作成: 2026-06-30

---

## 解決したい問題

算出ロジックで出た数値が業務に合わない場合、手動で上書きできるようにする。
さらに「やっぱり計算値に戻したい」ときのUIをどうするか。

---

## ゴースト値パターン（推奨）

「戻す」のためのモーダルは出さない。**計算値を常にゴーストで見せ続ける**ことで、ワンクリックで戻せる設計にする。

### 通常状態（自動計算中）

```
┌─────────────────────────────────────┐
│  労務費 原価                         │
│  ¥960,000  🤖 自動計算             │
│                           [修正する] │
└─────────────────────────────────────┘
```

### 「修正する」クリック → 確認モーダル

```
┌─────────────────────────────────────────┐
│  ⚠️  手動入力に切り替えます              │
│                                         │
│  この値は工程表・出面データから          │
│  自動的に計算されています。             │
│                                         │
│  手動入力に切り替えると、               │
│  以降の出面変更があっても               │
│  この値は自動更新されません。           │
│                                         │
│     [キャンセル]  [手動入力に切り替え]  │
└─────────────────────────────────────────┘
```

### 手動入力状態（Lock済み）

```
┌────────────────────────────────────────┐
│  労務費 原価                            │
│  ┌──────────────────────────────────┐  │
│  │  ¥ [  800,000  ] ✏️ 手動入力    │  │  ← 入力フィールド
│  └──────────────────────────────────┘  │
│  計算値: ¥960,000  [↩ 計算値に戻す]   │  ← ゴーストで表示し続ける
└────────────────────────────────────────┘
```

### 「↩ 計算値に戻す」クリック → 確認なしで即時戻す

**戻すときはモーダルを出さない。**
- 「修正する」は重い操作（モーダルあり）
- 「戻す」は軽い操作（即時・undo的感覚）
- この非対称性がデジタル弱者には安心感になる

戻した瞬間の表示:
```
┌─────────────────────────────────────┐
│  労務費 原価                         │
│  ¥960,000  🤖 自動計算             │  ← 計算値に戻った
│                           [修正する] │
└─────────────────────────────────────┘
```

## ゴースト値の更新ルール

手動Lock中も、バックグラウンドでは計算を続ける。

```
手動Lock中:
  表示値    = 手動入力値（変わらない）
  ゴースト  = 最新の計算値（リアルタイム更新）

→ 出面が増えるとゴーストの数字が変わる
→ ゴーストを見て「あ、やっぱり計算値の方が合ってきた」と気づける
```

差が大きくなったらアラートを出す（オプション）:
```
⚠️ 計算値（¥1,100,000）と手動入力値（¥800,000）の差が
   30万円以上になっています。確認をおすすめします。
```

## 「出力数値がどこから来るかわかる」ホバー設計

🤖 バッジにホバーすると計算内訳をポップアップ:

```
🤖 自動計算 にホバー →
┌──────────────────────────────────────────┐
│  計算内訳（クリックで工程表を確認）       │
│                                          │
│  田中 太郎: 12日 × ¥6,500 = ¥78,000    │
│  山田 花子:  8日 × ¥5,000 = ¥40,000    │
│  鈴木 一郎:  5日 × ¥7,000 = ¥35,000    │
│  ...                                     │
│  ────────────────                        │
│  合計: ¥960,000                          │
│                                          │
│  ※ 工程表のアサインと単価から算出        │
│  ※ 残業割増: OFF（会社設定）             │
│                                          │
│  [工程表を開く]  [出面を開く]            │
└──────────────────────────────────────────┘
```

**「どこをいじれば数字が変わるか」をポップアップの中に入れる。**

## 階層（大工程 / 小工程）の手動Lock挙動

```
大工程「大工工事」← 子の合計（手動入力不可・常に自動集計）
  └── 小工程A    ← ここだけ手動入力可
  └── 小工程B    ← ここだけ手動入力可
```

大工程は手動入力ボタンを非表示にする（グレーアウトではなく、そもそも出さない）。
代わりに「内訳を見る」ボタンで子工程に展開できるようにする。

---

## バッジ一覧（デザイン統一）

| バッジ | 色 | 意味 |
|---|---|---|
| 🤖 自動計算 | グレー | システムが算出した値 |
| ✏️ 手動入力 | オレンジ | ユーザーが直接入力した値 |
| 📄 OCR（確認待） | 黄色 | OCR読み取り済み・要確認 |
| 📄 OCR（確認済） | グリーン | OCR読み取り・確認完了 |

OCR確認待ちの行は背景色を薄黄色にしてリスト上で目立たせる。

## スマホ対応（現場監督）

スマホでは「ゴースト値 + 戻すリンク」を折りたたむ:

```
┌────────────────────────────────┐
│  労務費 原価                    │
│  ¥800,000  ✏️                  │
│  [計算値と比較] [↩ 戻す]       │
└────────────────────────────────┘
```

「計算値と比較」タップでゴースト値展開。
PC版との差はあるが、スマホ画面幅の制約上これが限界。
