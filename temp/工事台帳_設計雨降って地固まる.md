# 工事台帳 設計確定事項
作成: 2026-07-02（もやもや整理から確定事項を抽出・統合）

## 1. コアコンセプト

工事台帳は **「データ集約ビュー」**。入力画面ではない。

```
careecon_work（現場）          keiei_plus（工事台帳）
  出面・進捗・勤怠  ──────→   集約・確認・承認
  工程表・EVM      ──────→         ↓
  残業管理         ──────→   粗利がはじめて見える
Documents（見積・請求書）──→         ↓
                              財務記録として確定
```

- ユーザーが触るのは自動データが実態と合わないときだけ
- **工事台帳 API は keiei_plus に実装**（careecon_work に逆接続すると循環依存になるため NG）
- バッジは使わない。状態はセル背景色（薄紫=仮設定、薄青=計画値代替）+ ホバーで内訳表示

## 2. 画面設計

### 2-1. 総額タブ

**上部カード（工事全体サマリー）**

| 項目 | 定義 |
|---|---|
| 販売価格 | 受注額（手入力） |
| 出来高原価 | 進捗% × 見積原価（EV 理論値。参考表示） |
| 累積請求原価 | 届いた請求書の合計（AC 実額） |
| 粗利 | **販売価格 − 累積請求原価** |
| 粗利率 | 粗利 ÷ 販売価格 |

**費目テーブル**

```
費目    │  単価              │  見積原価    │  出来高原価  │  累積請求原価  │  差異
材料費  │  ¥3,200/m²        │  ¥3,000,000  │  ¥2,400,000  │   ¥2,200,000  │ +¥800K
外注費  │  一式              │  ¥8,000,000  │  ¥6,400,000  │   ¥5,500,000  │ +¥2.5M
労務費  │  ¥7,400/日（平均） │  ¥5,000,000  │  ¥3,200,000  │      −        │ −
経費    │  —                │    ¥500,000  │    ¥320,000  │     ¥200,000  │ +¥300K
────────┼────────────────────┼──────────────┼──────────────┼───────────────┤
合計    │                   │ ¥16,500,000  │ ¥12,320,000  │  ¥11,100,000  │
```

- 差異 = 見積原価 − 累積請求原価
- 労務費の累積請求原価 = −（自社社員コストのため請求書なし）
- バッジ列なし

### 2-2. 月次ビュー

```
月次で表示する:   出来高原価（EV）← 進捗% × 見積原価で自動計算
月次で出さない:   累積請求原価    ← 請求書の月次凸凹を避けるため
```

月次展開はデフォルト折り畳み。トグルで展開。  
材料費はデフォルト除外（チェックボックスで追加）。

### 2-3. 証憑バッジ・サイドパネル

テーブル上部に工事全体で1つのバッジ（「証憑 N 件」）。クリックで右からサイドパネルがスライドイン。

```
証憑登録フロー:
  PDF 投入 → 「どの明細行に紐づけますか？」← この 1 問だけ
          → 指定した明細の累積請求原価に加算
          → 月への配賦: doc_date が自動で決める
```

### 2-4. 修正ロジック

```
明細（個別）   → 編集可（ユーザーが触るのはここだけ）
費目合計       → 読み取り専用（明細の自動集計）
総額           → 読み取り専用（費目合計の自動集計）
```

### 2-5. 粗利連動 UI（見積/シミュレーション段階）

見積原価・粗利率・販売価格の 3 値のうち、最後に触ったフィールドが主となり残り 1 つを自動計算。

```
原価変更  → 粗利率が更新される
価格変更  → 粗利率が更新される
粗利率変更 → 販売価格が逆算される
```

DB 保存は `ledger_overrides.manual_value`（target_type: 'margin_rate'）のみ。  
実績フェーズ（施工中〜完工後）は粗利 = 販売価格 − 累積請求原価 の自動計算に切り替わる。

## 3. 損益のライフサイクル

| フェーズ | 販売価格 | 累積請求原価 | 粗利 |
|---|---|---|---|
| 見積段階 | 手打ち | 見積原価（手打ち） | シミュレーション |
| 施工中 | 確定 | 請求書が届くたびに積み上がる（途中） | 実績粗利（途中・正常） |
| 完工後 | 確定 | 全請求書確定 | 最終損益 |

工事中の「粗利が良く見える」は正常状態。請求書（売上・原価ともに）は後から出す/来るもの。

## 4. EVM 計算ロジック

### 4-1. 変数定義

| 変数 | 意味 | ソース |
|---|---|---|
| `p` | 工程（large_process） | careecon_work |
| `u` | ユーザー（工程アサイン済み） | process_users |
| `m` | 対象月 | — |
| `plan_pct[p][m]` | 月末時点の計画進捗% | baseline 保存（🚨 JSON 時系列必須） |
| `actual_pct[p][m]` | 月末時点の実績進捗% | `large_processes.progress`（手動入力） |
| `unit_price[u]` | 日当（月給 ÷ 稼働日数） | `project_users.unit_price` |
| `days[p][u]` | 工程期間内の計画人工数 | 工程 start〜end から導出 |
| `baseline_cost[p]` | 着工時の凍結労務費 | `process_baselines`（新規テーブル） |
| `estimated_cost[d]` | 発注金額 | keiei_plus `estimated_cost` Document |
| `actual_cost[d]` | 請求書金額 | keiei_plus `cost` Document |
| `delivery_planned[d]` | 納入予定日 | `documents.delivery_date_scheduled` |
| `delivery_actual[d]` | 納入実績日 | `documents.delivery_date_actual` |

### 4-2. 費目別 PV / EV / AC 計算式

**労務費**

```
baseline_cost[p] = Σ(u in p_at_start) unit_price[u] × days[p][u]  ← 着工時凍結

PV[p][m] = baseline_cost[p] × plan_pct[p][m]
EV[p][m] = baseline_cost[p] × actual_pct[p][m]
AC[p][m] = Σ(u in p_current) unit_price[u] × days[p][u] × actual_pct[p][m]
```

CV は構造上 ≈ 0（人員変動がなければ EV = AC の恒等式）。  
SV（進捗遅延）が労務費の主指標。CV は人員変動検知のみに使う。

**材料費**（EV はデフォルト OFF・チェックで追加）

```
PV[d][m] = estimated_cost[d]   if delivery_planned[d] が月 m 内、else 0
EV[d][m] = estimated_cost[d]   if delivery_actual[d] が月 m 内（チェック ON 時のみ）
AC[d][m] = actual_cost[d]      if delivery_actual[d] が月 m 内（請求書到着）
           estimated_cost[d]   if delivery_actual[d] が月 m 内かつ請求書未着（暫定）
           0                   otherwise
```

**外注費**（出来高按分・PL 方式）

```
PV[d][m] = estimated_cost[d] × plan_pct[p][m]      ← p = 外注 d が紐づく工程
EV[d][m] = estimated_cost[d] × actual_pct[p][m]
AC[d][m] = actual_cost[d]    × actual_pct[p][m]    ← 請求書到着後（確定）
           estimated_cost[d] × actual_pct[p][m]    ← 請求書未着（暫定）
```

### 4-3. 月次増分変換

```
monthly_pct[p][m] = actual_pct[p][m] - actual_pct[p][m-1]

EV 月次増分[p][m] = baseline_cost[p] × monthly_pct[p][m]
AC 月次増分[p][m] = Σ(u) unit_price[u] × days[p][u] × monthly_pct[p][m]
```

### 4-4. EV スコープ

```
デフォルト: 労務費 EV + 外注費 EV のみ
材料費追加:  チェックボックスで ON（高額になりやすくグラフ縮尺が変わる）
スコープ切替: EV と PV を同時に切り替える（SV の整合性を保つ）
```

### 4-5. フォールバック

| データ状況 | フォールバック |
|---|---|
| 工程表なし | 工期を均等割りして plan_pct を線形補完 |
| actual_pct 未入力 | 0（未着手扱い） |
| unit_price 未設定 | ¥17,500/日（月収 35 万円 ÷ 20 日）← 薄紫背景で表示 |
| delivery_actual 未設定 + delivery_planned 未来 | EV = 0 |
| delivery_actual 未設定 + delivery_planned 過去 | EV = estimated_cost（計画日に納入とみなす） |
| delivery_actual も delivery_planned も未設定 | EV = 0 |
| estimated_cost なし | large_process.unit_price で代替 |

### 4-6. ベースライン凍結

**トリガー**: 案件ステータス「着工前 → 施工中」に変わった瞬間に 1 回のみ実行。

**保存内容**（`process_baselines` テーブル・新規）:
- 各工程の `baseline_cost`（着工時アサイン人員 × 計画人工数の合計）
- 各工程の `plan_pct_series`（月 → 進捗% のマップを JSON 保存）← 🚨 **必須。なければ PV が計算できない**
- 凍結日時

工程未完成のまま着工 → 工期均等割りで仮生成。ベースライン再生成は admin のみ・初回のみ可。

## 5. 外注費 EV の紐づきフォールバック（3 段階）

```
Tier 1: document.large_process_id あり
  → その工程の actual_pct で計算（最精度）

Tier 2: keiei_plus customer ↔ careecon_work company の対応が確立済み
  → 対応する工程群の平均 actual_pct で計算（Phase2 以降）

Tier 3（MVP 現状）: 両方なし
  → プロジェクト全体の加重平均 actual_pct で計算
```

**ゲスト未登録時の自動登録**（waiting → working トリガー）:

案件にゲストが参加しているが ProcessUser が未登録の場合、工事開始時に全工程へ自動登録。  
`find_or_create_by` で冪等。自動登録レコードには `source: :auto_initialized` フラグ。  
UI: 薄紫背景 + ホバー「工事開始日に自動登録されました。工程に紐づけると精度が上がります [工程表へ →]」

**ProcessUser 自動登録のトリガー分担:**
- メンバー追加時 after_create（もやもや⑤）= 全ユーザー対象・随時
- 工事開始時 waiting→working（もやもや⑫）= ゲストのみ・補完

## 6. データ登録フロー

### Lock & Override

```
基本: 自動計算
  ↓ 違うと感じたら「手動で修正」
手動入力モード（自動更新除外 → ホバーで「手動入力値」表示）
  ↓「計算に戻す」で自動計算に復帰
```

保存先: keiei_plus `ledger_overrides` テーブル（新規）。  
「計算に戻す」= そのレコードを destroy するだけ。追加ロジック不要。

### OCR 入口

一次分類: **支払い / 入金 の 2 択**（左右に大きく表示）  
二次分類: 見積 or 請求書 → OCR 後の確認画面で 1 クリック選択

### 配賦ロジック

月次配賦の精度は重要でない。中小建設業者にとって大事な順:

```
1. 工事ごとの総額損益（最重要）
2. キャッシュフロー
3. 月次配賦の精度（どうでもいい）
```

月次の出来高原価は 出来高% × 見積原価 で自動配賦。請求書を月次に振り分ける操作は不要。

## 9. 残業・割増ロジック

**MVP: 割増計算はデフォルト ON・会社設定で OFF**

> ⚠️ 労働基準法に基づき、時間外・深夜・休日労働には割増賃金の支払いが義務付けられています。

```
会社設定 > 労務費計算
  [ON 推奨] 時間外割増 (25%) ※法定
  [ON 推奨] 深夜割増  (25%) ※法定
  [ON 推奨] 休日割増  (35%) ※法定
  [ON 推奨] 月 60h 超割増 (25%) ※法定（中小特例あり）
  割増率は各項目で手動入力可（社内ルールに対応）
```
